# 🏷️ СИСТЕМА МЕТOK ДЛЯ РАЗРАБОТЧИКОВ

## 💡 ЦЕЛЬ

Предотвратить дублирование, конфликты и нарушение зависимостей при внесении изменений в код.

---

## 📊 ТИПЫ МЕТОК

### 1. ⚠️ CRITICAL — Критичная функция
**Когда использовать:** Функция которую нельзя ломать или менять без понимания последствий

**Примеры:**
- `loadMoreTagTracks()` — пагинация тегов
- `switchView()` — переключение видов
- `setupTagTracksInfiniteScroll()` — infinite scroll

**Где ставить:**
```javascript
// ⚠️ CRITICAL: Загрузка треков тега (пагинация)
// 🔒 LOCKED: Проверять isLoadingTagTracks перед вызовом!
async function loadMoreTagTracks() { ... }
```

---

### 2. 🔒 LOCKED — Заблокировано/Зависимости
**Когда использовать:** Есть зависимости от других функций/переменных

**Примеры:**
- Зависимость от флагов (`isLoadingTagTracks`)
- Порядок вызова (`selectTag()` → `setupTagTracksInfiniteScroll()`)
- Цепочки вызовов

**Где ставить:**
```javascript
// 🔒 LOCKED: Вызывать ОДИН раз в selectTag(), НЕ пересоздавать!
// Зависимости: tagTracksObserver, tagTracksHasMore, isLoadingTagTracks
function setupTagTracksInfiniteScroll() { ... }
```

---

### 3. 💡 ПОДСКАЗКА — Комментарий разработчику
**Когда использовать:** Полезная информация для будущего изменения

**Примеры:**
- Ссылка на документацию
- Предупреждение о проблемах
- Рекомендации по использованию

**Где ставить:**
```javascript
// script.js
// 💡 ПОДСКАЗКА ДЛЯ РАЗРАБОТЧИКА:
// - Перед изменением проверь docs/CODE_CHEATSHEET.md
// - Критичные функции помечены ⚠️ CRITICAL и 🔒 LOCKED
```

## 📝 ФОРМАТ КОММЕНТАРИЕВ

### В начале файла (header)
```javascript
// filename.js
//
// 💡 ПОДСКАЗКА ДЛЯ РАЗРАБОТЧИКА:
// - Краткое описание что делает файл
// - На что обратить внимание
// - Ссылка на документацию
```

### Перед функцией
```javascript
// ⚠️ CRITICAL: Назначение функции
// 🔒 LOCKED: Важное предупреждение
// Зависимости: var1, var2, function3
// Цепочка: A() → B() → C()
function myFunction() { ... }
```

### Внутри функции (warning)
```javascript
function myFunction() {
    // ⚠️ WARNING: Здесь возможна проблема X
    someRiskyCall(); 
}
```

## 🎯 ПРАВИЛА ИСПОЛЬЗОВАНИЯ

### ✅ ДЕЛАЕМ

1. **Ставим метки НА ВСЕ критичные функции**
   - Пагинация
   - Infinite scroll
   - Переключение видов
   - Восстановление состояния

2. **Указываем ЗАВИСИМОСТИ явно**
   ```javascript
   // Зависимости: flag1, variable2, function3
   ```

3. **Пишем ЧТО будет если нарушить**
   ```javascript
   // ❌ Не вызывать здесь — приведёт к дублированию
   ```

4. **Ссылаемся на документацию**
   ```javascript
   // 📚 Docs: docs/CODE_CHEATSHEET.md#infinite-scroll
   ```

5. **Обновляем CODE_CHEATSHEET.md**
   - Добавляем новую метку в таблицу
   - Обновляем раздел "Критичные зависимости"

---

### ❌ НЕ ДЕЛАЕМ

1. **Не ставим метки на простые функции**
   ```javascript
   // Плохо:
   // ⚠️ CRITICAL
   function escapeHtml(text) { ... }
   
   // Хорошо: ставим только на сложную логику
   ```

2. **не используем слишком много меток**
   - Если всё "критично" — значит ничто не критично
   - Максимум 10-15% функций с метками

3. **Не пишем очевидные вещи**
   ```javascript
   // Плохо:
   // ⚠️ CRITICAL: Эта функция рендерит HTML
   function render() { ... }
   
   // Хорошо:
   // ⚠️ CRITICAL: Рендерит ТОЛЬКО новые элементы
   // ❌ Не рендерить весь список — будет мерцание
   ```

## 📋 ЧЕКЛИСТ ПЕРЕД ДОБАВЛЕНИЕМ МЕТКИ

Перед тем как поставить ⚠️ CRITICAL:

- [ ] Эта функция действительно критична?
- [ ] Есть ли зависимости которые можно нарушить?
- [ ] Будет ли проблема если функцию вызвать не там?
- [ ] Поймёт ли новый разработчик что здесь важно?

Если хотя бы один ответ "Да" — ставим метку!

## 🔍 ПРИМЕРЫ ИЗ ПРОЕКТА

### Пример 1: Infinite Scroll
```javascript
// ⚠️ CRITICAL: Infinite Scroll для Tags view
// 🔒 LOCKED: Вызывается ОДИН раз в selectTag(), НЕ пересоздавать!
// Зависимости: tagTracksObserver, tagTracksHasMore, isLoadingTagTracks
let tagTracksObserver = null;
function setupTagTracksInfiniteScroll() {
    // Удаляем старый observer
    if (tagTracksObserver) {
        tagTracksObserver.disconnect();
        tagTracksObserver = null; // ← Важно обнулить!
    }
    // ...
}
```

**Почему критично:** Пересоздание observer приводит к дублированию вызовов

**Что будет если нарушить:** Теги загружаются 2-3 раза одновременно

---

### Пример 2: Рендеринг
```javascript
// ⚠️ CRITICAL: Рендеринг страницы треков
// 🔒 LOCKED: НЕ вызывать setupTagTracksInfiniteScroll() здесь!
// Вызывать только updateSentinel text после рендера
function renderTagTracksPage(pageTracks) {
    // ... рендер ...
    
    // ⚠️ WARNING: Пересоздание observer!
    setupTagTracksInfiniteScroll(); 
}
```

**Почему критично:** Неправильное место для инициализации observer

**Что будет если нарушить:** Утечки памяти, дублирование вызовов

---

### Пример 3: Восстановление состояния
```javascript
// ⚠️ CRITICAL: Восстановление состояния плеера для Tags
// 🔒 LOCKED: Вызывать ПОСЛЕ selectTrack() для сохранения позиции
// Зависимости: tagTracks, currentTag, currentTrackIndex
function restoreTagTrackPlayer(track, currentTime) {
    // ...
}
```

**Почему критично:** Порядок вызова важен для корректного состояния

**Что будет если нарушить:** Плеер показывает не тот трек
