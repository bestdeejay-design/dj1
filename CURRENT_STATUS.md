## 🎯 ПРАВИЛЬНЫЙ ПОДХОД (Data-Driven)

### Методология:

```
ВСЕГДА ИДТИ ОТ ДАННЫХ:

1. Взять ВСЕ промты из API
   ↓
2. Составить список потенциальных тегов
   ↓
3. Проверить каждый тег в промтах
   ↓
4. Фильтр: тег должен быть ≥30 раз
   ↓
5. Только потом добавлять в tags-data.json
```

### Почему это правильно:

**❌ БЫЛО (неправильно):**
- Придумал тег "Phonk" → нашёл 0 треков
- Придумал тег "PluggnB" → нашёл 0 треков
- Придумал тег "Amapiano" → нашёл 2 трека

**✅ СТАЛО (правильно):**
- Нашёл в промтах "techno" 1,234 раза → добавил тег
- Нашёл в промтах "energetic" 1,567 раз → добавил тег
- Нашёл в промтах "amapiano" 2 раза → пропустию (<30)

## 📋 СЛЕДУЮЩИЕ ШАГИ

### Когда анализ завершится:

**1. Изучить результаты**
```bash
tail -100 phase1-analysis-output.txt
cat data/api-analysis-results.json | python3 -m json.tool | less
```

**2. Сравнить с текущими тегами**
```bash
python3 compare_tags.py  # покажет чего не хватает
```

**3. Принять решения**
- Какие теги добавить (найдены ≥30 раз, нет в current)
- Какие теги удалить (<30 треков или 0 треков)
- Какие категории расширить

**4. Обновить tags-data.json**
```bash
python3 update_tags_from_api.py
```

**5. Протестировать**
```bash
# Открыть index.html
# Проверить новые таги
# Проверить кликабельность
```

**6. Закоммитить**
```bash
git add data/tags-data.json
git commit -m "feat: Data-driven tag expansion"
git push
```
